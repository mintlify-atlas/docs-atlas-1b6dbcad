## Your First Worktree

This guide walks you through creating and managing your first Git worktree using `wt`.

<Note>
  Make sure you've completed the [installation](/installation) before proceeding.
</Note>

### Navigate to Your Repository

First, change to any Git repository where you want to use worktrees:

```text
cd ~/your-project
```

<Warning>
  You must run `wt` commands from within a Git repository. The tool will automatically detect the repository root.
</Warning>

### Create a New Worktree

Create a new worktree for a feature branch:

```bash
wt new feature/my-feature dev
```

This command:
1. Creates a new branch `feature/my-feature` based on `dev`
2. Sets up a worktree in `~/worktrees/<repo-name>/feature-my-feature`
3. Copies `.env` files from the main worktree
4. Auto-detects and installs dependencies (npm/yarn/pnpm/bun)
5. Changes your directory to the new worktree

<CodeGroup>
```bash Terminal Output
$ wt new feature/my-feature dev
[info] Creating branch 'feature/my-feature' from 'dev'
Preparing worktree (new branch 'feature/my-feature')
HEAD is now at a1b2c3d Latest commit message
[ok] .env copied to /home/user/worktrees/my-project/feature-my-feature/.env
[info] Installing dependencies with pnpm in /home/user/worktrees/my-project/feature-my-feature
Progress: resolved 245, reused 245, downloaded 0, added 0
Dependencies installed in 2.1s
[ok] Worktree ready: /home/user/worktrees/my-project/feature-my-feature
```

```bash Working with Existing Branch
$ wt new existing-branch
[info] Adding existing branch 'existing-branch' to worktree
[ok] .env copied to /home/user/worktrees/my-project/existing-branch/.env
[info] Installing dependencies with npm in /home/user/worktrees/my-project/existing-branch
added 245 packages in 3.2s
[ok] Worktree ready: /home/user/worktrees/my-project/existing-branch
```
</CodeGroup>

<Tip>
  If you omit the base branch, `wt` uses your current branch as the base:
  ```bash
  wt new feature/quick-fix
  ```
</Tip>

## List Your Worktrees

View all worktrees with an interactive selector:

```bash
wt ls
```

### Interactive Mode (default)

When `fzf` is installed, you'll see an interactive list:

```text
wt ls> _
  main           a1b2c3d   clean   /home/user/my-project
> feature/auth   b2c3d4e   dirty   /home/user/worktrees/my-project/feature-auth
  feature/ui     c3d4e5f   clean   /home/user/worktrees/my-project/feature-ui
  bugfix/login   d4e5f6g   clean   /home/user/worktrees/my-project/bugfix-login
  4/4
```

- Use arrow keys to navigate
- Press Enter to change directory to the selected worktree
- Press Esc to cancel

### Plain Text Mode

For scripting or when `fzf` is not available:

```bash
wt ls --plain
```

```text
BRANCH                         HEAD       STATE    PATH
------                         ----       -----    ----
main                           a1b2c3d    clean    /home/user/my-project
feature/auth                   b2c3d4e    dirty    /home/user/worktrees/my-project/feature-auth
feature/ui                     c3d4e5f    clean    /home/user/worktrees/my-project/feature-ui
bugfix/login                   d4e5f6g    clean    /home/user/worktrees/my-project/bugfix-login
```

## Switch Between Worktrees

Change directory to a specific worktree:

```bash
wt cd feature/auth
```

You can use:
- **Branch name**: `wt cd feature/auth`
- **Slugified name**: `wt cd feature-auth`
- **Directory name**: `wt cd feature-auth`

<Note>
  The `wt cd` command requires the shell function installed by `wt install`. It won't work if you run the script directly with `./wt cd`.
</Note>

### Interactive Selection

Omit the branch name to choose interactively:

```bash
wt cd
```

## Start Development Server

Run the dev server in any worktree:

```bash
# From within a worktree
wt dev

# Or specify a branch
wt dev feature/auth
```

The `wt dev` command:
- Auto-detects your package manager (bun/pnpm/yarn/npm)
- Runs the appropriate dev command (`bun run dev`, `pnpm dev`, etc.)
- Uses the correct `package.json` location

<CodeGroup>
```bash bun
[info] Running dev with bun in /home/user/worktrees/my-project/feature-auth
$ bun run dev
  VITE v5.0.0  ready in 234 ms
  ➜  Local:   http://localhost:5173/
```

```bash pnpm
[info] Running dev with pnpm in /home/user/worktrees/my-project/feature-auth
> dev
> next dev
  ▲ Next.js 14.0.0
  - Local:        http://localhost:3000
```
</CodeGroup>

## Remove a Worktree

Delete a worktree you no longer need:

```bash
wt rm feature/old-feature
```

### Interactive Removal

For safety, you can choose from a list and confirm:

```bash
wt rm
```

```text
wt rm> _
> feature/auth   b2c3d4e   dirty   /home/user/worktrees/my-project/feature-auth
  feature/ui     c3d4e5f   clean   /home/user/worktrees/my-project/feature-ui
  bugfix/login   d4e5f6g   clean   /home/user/worktrees/my-project/bugfix-login

Really remove 'feature/auth' (/home/user/worktrees/my-project/feature-auth)? [y/N]: y
[warn] Dirty working tree detected: /home/user/worktrees/my-project/feature-auth
[ok] Removed worktree: /home/user/worktrees/my-project/feature-auth
```

### Force Removal

Skip confirmation and remove dirty worktrees:

```bash
wt rm -f feature/old-feature
```

<Warning>
  The primary worktree (usually `main` or `master`) cannot be removed.
</Warning>

## Sync Environment Files

Keep `.env` files synchronized across all worktrees:

```bash
# Sync to all worktrees
wt sync

# Sync to specific worktree
wt sync feature/auth
```

This copies `.env` from your primary worktree to all other worktrees.

## Check Status Across Worktrees

View git status for all worktrees at once:

```bash
wt status
```

```text Terminal Output
  clean

```

## Common Workflows

### Starting New Feature Work

```bash
# Create and switch to new feature branch
wt new feature/user-settings main

# Install dependencies and run dev server (automatic)
wt dev
```

### Switching Context

```bash
# List and select worktree interactively
wt ls

# Or jump directly
wt cd feature/other-feature
```

### Cleaning Up

```bash
# Check what's dirty
wt status

# Remove completed feature branches
wt rm feature/completed-feature

# Or choose interactively
wt rm
```

## Package Manager Detection

`wt` automatically detects your package manager based on lockfiles:

| Lockfile | Package Manager | Install Command | Dev Command |
|----------|----------------|-----------------|-------------|
| `bun.lockb` | bun | `bun install` | `bun run dev` |
| `pnpm-lock.yaml` | pnpm | `pnpm install --frozen-lockfile` | `pnpm dev` |
| `yarn.lock` | yarn | `yarn install --frozen-lockfile` | `yarn dev` |
| `package-lock.json` | npm | `npm ci` | `npm run dev` |
| *(none)* | npm (fallback) | `npm install` | `npm run dev` |

## Worktree Locations

By default, worktrees are stored in:

```text
~/worktrees/<repository-name>/<branch-slug>/
```

For example:
```text
~/worktrees/my-project/
├── feature-auth/
├── feature-ui/
└── bugfix-login/
```

Branch names with slashes are slugified (e.g., `feature/auth` → `feature-auth`).
